# gitHub_Binggozz_vje-ba_jedan

## Samostalan zadatak

## 1. Sto je Visalu Studio code i za sto se najcesce koristi?
- Visual Studio Code je jednostavan i jako versatibilan program za kodiranje jer preko njega mozemo dodati razne plugine i stvari. 

## 2. Kako se zove panel u kojem vidis sve datoteke i mape svog projekt?
- File Explorer

## 3. Kako mozes instalirati ekstenzije u VS Codeu i cemu one sluze?
- Odemo na tab Extensions i instaliramo. One sluze za pomaganje oko zadataka.

## 4. Kako otvoriti postojeci projekt ili mapu u VS Code-u?
- Odemo na file i onda kliknemo open file pa nam se otvori File Explorer i tamo nademo projekt ili mapu na kojoj zelimo raditi.

## 5. Sto se sve moze raditi u Exploreru? 
- Mozemo pogledati strukturu projekta, stvoriti i maknuti datoteke i mape.

## 6. Kako mozes brzo pronaci odredenu datoteku u projektu?
- Ctrl + P

## 7. Kako mozes istovremeno gledati dvije razlicite datoteke u editoru?
- Ctrl + Shift + P pa se otvori panel i onda izaberemo Split Editor

## 8. Kako pokreces automatski formatiranje koda u VS Code-u?
- Ctrl + Shift + P i onda Format Document

## 9. Sto je version control i zasto se koristi?
- Softver koji sluzi za pracenje promjena u kodu i koordinaciju rada vise clanova tima

## 10. Kako se VS Code-u otvara panel za Git (Source Control)?
- Sa lijeve strane u traci imamo ikonicu za Source Control pa mozemo to kliknut

## 11. Sto znace commit, push, pull?
- Commit - mijenja verziju projekta kojeg imamo na racunalu
- Push - mjenjamo verziju koda na cloud-u
- Pull - Uzimamo promjene za vezije na cloud-u i stavljamo na nasu kopiju na racunalu

## 12. Cemu sluzi commit poruka i kako treba izgledati?
- Naredba Git commit pohranjuje kopije promjena iz vašeg radnog direktorija u vaše Git spremište
- ### Primjer:  git reset ahead
  ###           git commit --ammend
  ###
  ###           git rebase
  ###           git diff
  ###           git log -f
  ###           git grep [nešto]
 
## 13. Kako mozes idjeti koje si promjene napravio u pojedinoj datoteci?
- Možemo kroz Version Control Panel u VSCodu ili na GitHub-u tako da gledamo starije verzije i commit message

## 14. Kreiraš novu datoteku. Koji su koraci da se promjena pošalje na GitHub?
- Prvo commitamo na lokalnu verziju i onda pushamo na repository na cloudu

## 15. Kako se vraćaš na stariju verziju datoteke ako si pogriješio?
- Koristimo git reset

## 16. Na koji način možeš provjeriti status projekta i koje datoteke su izmijenjene?
- Koristimo git status
